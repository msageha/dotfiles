# Fish Shell

- Functions: Use named functions with proper argument handling.
- Variables: Utilize scope modifiers (-g, -l) appropriately.
- Path Handling: Use path manipulation functions instead of string operations.
- Completions: Implement tab completions for custom commands.
- Error Handling: Check status after critical commands.
- Command Substitution: Use (command) syntax consistently.
- Conditionals: Utilize fish-specific if/test syntax.
- Loops: Implement for loops with proper iteration techniques.
- Environment: Set and export variables properly.
- Aliases: Use functions instead of aliases for complex operations.
- Comments: Document function purpose and usage.
- Input Validation: Validate and sanitize user inputs.
- Configuration: Store settings in config.fish or dedicated conf.d files.
- Use fish-specific features (like switch statements and event handlers).
- Include usage/help information and proper error messages.
